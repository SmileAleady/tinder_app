# Tinder 卡片拖动交互功能

## 功能描述

实现了完整的 Tinder 风格用户卡片拖动交互系统，具有以下特性：

### 1. 卡片拖动效果
- 用户可以水平拖动用户卡片
- 拖动过程中卡片实时跟随手指位置
- 支持从左侧或右侧拖动

### 2. 按钮动态缩放
- **不喜欢按钮（红叉）**：向左拖动时放大，最大可放大至 2.5 倍
- **其他按钮**：向左拖动时缩小，最小缩小至 0.3 倍
- 达到 30% 时按钮开始淡出
- 松手后按钮自动恢复原状

### 3. 下一个用户卡片生成
- 拖动过程中，如果拖动距离超过屏幕宽度的 50%，自动在底部生成下一个用户卡片
- 下一个卡片以动画形式出现（缩放 + 透明度过渡）

### 4. 卡片划出逻辑
- **超过 50% 阈值**（一半面积超出屏幕）
  - 松手后卡片从左侧划出屏幕
  - 显示下一个用户卡片
  - 所有按钮恢复到原始状态
  - 自动加载新用户

- **未超过 50% 阈值**（一半面积未超出屏幕）
  - 松手后卡片平滑回弹到中间位置
  - 所有按钮恢复到原始状态
  - 继续显示当前用户

## 技术实现

### 核心组件

| 组件 | 作用 |
|------|------|
| `GestureDetector` | 检测水平拖动手势 |
| `Transform.translate()` | 实现卡片拖动位置变化 |
| `AnimationController` | 管理卡片划出和回弹动画 |
| `ScaleTransition` | 下一个卡片出现动画 |
| `Transform.scale()` | 按钮放大/缩小效果 |

### 关键方法

| 方法 | 功能 |
|------|------|
| `_handleDragUpdate()` | 处理拖动中的实时更新 |
| `_handleDragEnd()` | 处理拖动结束的逻辑判断 |
| `buttonScale` getter | 计算普通按钮的缩放比例 |
| `rejectButtonScale` getter | 计算不喜欢按钮的缩放比例 |

## 用户交互流程

```
用户长按卡片
    ↓
拖动卡片
    ├─ 显示下一个卡片（距离 > 50%）
    └─ 按钮动态缩放
    ↓
松手
    ├─ 距离 > 50% → 卡片划出 → 加载新用户
    └─ 距离 ≤ 50% → 卡片回弹 → 保持当前用户
```

## 性能优化

- 使用 `TickerProviderStateMixin` 确保流畅的动画
- AnimationController 与 State rebuild 协同工作
- 通过 `setState()` 精确控制重绘区域
